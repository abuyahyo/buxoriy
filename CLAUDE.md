# Buxoriy — Имом Бухорий «Саҳиҳ» тўплами

Имом Бухорийнинг «Ал-Жомиъ ас-Саҳиҳ» ҳадис тўпламининг ўзбекча таржимаси. Статик веб-сайт: https://abuyahyo.github.io/buxoriy/

## Тузилма

- `index.html` — бир-файлли веб-илова (HTML + CSS + JS). Икки босқичли юклаш: `index.json` (енгил метамаълумот) бош ва китоб саҳифаларини зудлик билан кўрсатади, `data.json` (тўлиқ) орқа фонда юкланади.
- `data.json` — барча китоб/боб/ҳадислар (минификация қилинган JSON, индентациясиз). ~7.6 МБ.
- `index.json` — китоб/боб метамаълумоти ва ҳадис сонлари (ҳадис контенти йўқ). `build_index.py` орқали `data.json` дан генерация қилинади. **Ҳар сафар `data.json` ўзгартирилгандан кейин қайта генерация қилиниши шарт.** ~1.6 МБ.
- `apple-touch-icon.png` — iPhone "Add to Home Screen" иконкаси (180×180)
- `favicon` — `index.html` ичида inline SVG

GitHub Pages автоматик `main` бранчдан хизмат кўрсатади.

### index.json генерацияси

`data.json` ўзгартирилгандан сўнг:
```bash
python3 build_index.py
```

`index.json` тузилмаси: `data.json` каби, фақат ҳар бир бобда `hadislar` массиви ўрнига `hC` (ҳадис сони) майдони. `izoh` ва `muallaqot` мавжуд бўлса сақланади.

## Таҳрирлаш услуби

Иш услуби: фойдаланувчи манба (PDF/расм/матн) юборади, мен ҳар бир боб/ҳадис ҳолатини солиштириб, фарқларни `data.json`-да янгилайман.

### Манба ва маълумот солиштириш

- Манбадаги матнни **айнан** кўчираман — имло, тиниш белгилари, сўз тартиби кофинча муҳим.
- Қуръон оятлари, илова бўлим, footnote, изоҳлар сақланади.
- Агар манбада боб номи бўш бўлса (масалан, фақат "X-БОБ"), `nomi` ҳам шундай бўш сақланади.

### Matn формати

- Параграфлар орасида `\n\n` (иккита).
- Айрим ҳолларда битта сатр кўчиш `\n` билан.
- Манбадаги "X. Narrator розияллоҳу анҳодан ривоят қилинади:" киришма ҳадис матнига кирмайди — у `rowi` майдонида сақланади.

### Rowi формати

- Оддий: `"Оиша розияллоҳу анҳо"`
- Занжир: `"Зайнаб — Умму Салама розияллоҳу анҳодан"` (Х уни Y дан ривоят қилади)
- Махсус: `"Умайр (Ибн Аббос розияллоҳу анҳумонинг мавлоси)"`

### Footnote (`*`) ва изоҳ

Манбада ҳадис матни ичида `*` белгиси билан изоҳга ишора қилинса:
- `*` белгиси `matn` ичида қолади (масалан, "хамийла*", "Журуфдаги*")
- Изоҳ матни ҳам `matn` охирига `\n\n* ...` шаклида қўшилади (битта footnote бўлса)
- Алоҳида "Изоҳ:" блоки бўлса, у `izoh` майдонига ёзилади

Мисол (322-ҳадис):
```
matn: "...жунубликда бир идишдан ғусл қилар эдик».\n\n* 298-ҳадисга қаранг."
izoh: "Бу ва бошқа бир неча ҳадисларда «нифос» деб..."
```

### Имло хоссалари

- "розияллоҳу анҳо/анҳу/анҳумо" — нинг шакли мос ровийга қараб ёзилади
- "соллаллоҳу алайҳи васаллам" — Расулуллоҳ номи билан
- "ҳ" (Уzб) ва "х" Кириллда фарқли — манбадаги сингари тутилади
- Қавс билан изоҳ: `(...)` — таржимон қўшимчаси
- Мураббаъ қавс: `[...]` — алтернатив таржима ёки нашрга оид

## JSON форматини сақлаш

`data.json` минификация қилинган (индентациясиз). Янгилашда қуйидаги Python шакли ишлатилади:

```python
with open('data.json', 'r', encoding='utf-8') as f:
    db = json.load(f)
# ... ўзгаришлар ...
with open('data.json', 'w', encoding='utf-8') as f:
    json.dump(db, f, ensure_ascii=False, separators=(',', ':'))
```

`indent=2` ИШЛАТИЛМАСИН — у бутун файлни форматлаб юборади ва диff жуда катта бўлади.

## Код тузилмаси (`index.html`)

Битта файл: `<head>` + inline CSS → 4 та `<div class="page">` блок → `<script>` JS.

### Саҳифалар (4 та)

Ҳамма саҳифа DOM'да доимий туради, бирваракай битта кўриниб туради — `showPage(id)` бошқаради.

- `pgHome` — китоблар грид + қидирув инпут + қидирув натижалари
- `pgBob` — танланган китобнинг боблари
- `pgHadisList` — танланган бобнинг ҳадислари
- `hadisPage` — битта ҳадис тўлиқ матн билан

### Glob state

```js
let DB = [];                  // data.json — массив
let ST = { page, kid, bid, hid };   // жорий навигация
let _searchState = { query, kitob[], bob[], hadis[], hadisShown };
let fs, th;                   // localStorage: фонт ўлчами, тема
```

### Hash routing

URL hash'и саҳифа ҳолатини сақлайди. Browser back/forward `applyHash()` орқали қайта тиклайди.

- `#k/1` — китоб
- `#k/1/b/2` — боб
- `#k/1/b/2/h/3` — ҳадис
- `#q=сўз` — қидирув натижалари (encodeURIComponent)

### App объекти асосий методлари

| Метод | Вазифаси |
|-------|----------|
| `goHome(_keepSearch)` | Бош саҳифа. Флаг билан қидирувни сақлаш мумкин |
| `openKitob(kid, scrollToBid?)` | Китоб бобларини кўрсатиш |
| `openBob(kid, bid, scrollToHid?)` | Бобнинг ҳадисларини кўрсатиш |
| `openHadis(kid, bid, hid)` | Ҳадис тўлиқ кўриниш |
| `doSearch(v, fromHash?)` | Қидирув: 3 та секцияда натижа (китоб/боб/ҳадис) |
| `_renderSearch()` | `_searchState`'дан DOM яратиш |
| `showMoreHadis()` | Кейинги 100 та ҳадисни кўрсатиш |
| `restoreSearch()` | Натижаларга қайтиш |
| `prevHadis()` / `nextHadis()` | Ҳадислар ўртасида ўтиш |
| `toggleTheme()` / `adjustFontSize()` / `copyHadis()` / `toggleIzoh()` | UI |

### Helper функциялар

- `getK(kid)`, `getB(kid,bid)`, `getH(kid,bid,hid)` — массивдан топиш
- `allH()` — ҳамма ҳадислар flat массиви (prev/next учун)
- `bobNomi(s)` — боб номини форматлаш (катта/кичик ҳарф, "Аллоҳ", "Набий" автоматик)
- `snippet(text, query)` — қидирувда сўз атрофида ~200 ҳарф
- `escHtml(s)`, `escRegex(s)` — XSS / regex ҳимояси
- `syncSearchToHash(q)` — қидирувни URL'га чиқариш (push/replace)
- `homeBcItems()` — breadcrumb'нинг бошланиши («Бош саҳифа» + ихтиёрий «"X" қидирув» линки)
- `bc(containerId, items)` — breadcrumb HTML қуриш

### CSS наименование

- `.kitob-*` — китоб карталари (top, num, name, stats)
- `.bob-*` — боб карталари (card, name, izoh, count, muallaqot-badge)
- `.hadis-*` — ҳадислар (item, text, izoh, narrator, source, tb-btn)
- `.result-*` — қидирув натижалари (section, item, label, text, where, more-btn)
- `.bc-*` — breadcrumb (trail, link, sep, cur)
- `.fab-*` — флоат тугмалари (nav, theme)
- CSS variables `:root` ва `.light` — тема рангларини бошқаради

### Янги функция қўшиш йўриғи

- Янги саҳифа: HTML'да янги `<div id="pgXxx" class="page">` қўшиш, `showPage()`'нинг массивига киритиш, App.openXxx() метод яратиш, `applyHash()`'да URL pattern қўшиш.
- Янги қидирув майдони: `doSearch()` ичида `inX = ...includes(q)` қўшиш, `_renderSearch()`'да label ўзгартириш.
- Янги breadcrumb элементи: керакли open* методда `bc(...)` чақирувида item қўшиш.

## Workflow

1. Фойдаланувчи бир неча боб/ҳадис учун манба расмларини юборади (одатда 2-3 боб бирга).
2. `data.json`-нинг шу қисмини ўқиб, манба билан солиштираман.
3. Фарқли жойларни Python скрипти билан янгилайман.
4. Verify қилиб, `git diff --stat` бир сатрлик ўзгаришни кўрсатиши керак.
5. Commit + push `main` га (GitHub Pages 1-2 дақиқада янгилайди).
